Mezzanine in a nutshell

  • A toolkit, built on top of Django, designed for building sites with editable content
  • A toolkit with a somewhat opinionated base template ... that you are free to discard if you don't like it
  • BSD Licence
  • Supports Django 1.7, 1.8
  • Supports Python 2.6, 2.7, 3.3+
  • Stephen McDonald
  • mezzanine.jupo.org
  • toolkit, distinct from CMS. Set of APIs that you put together rather than a full suite you can configure
  • opinionated in the same way that the Django default template is. Useful to start with
  • Versions are for latest stable, 4.0.1 at time of writing. 3.1.x series supports Django 1.6
  • 2-clause BSD

Getting started


apt-get install python3 python3-dev virtualenvwrapper \
	libjpeg8 libjpeg8-dev
mkvirtualenv --python=`which python3` my_first_mezzanine_site
pip install mezzanine
mezzanine-project my_first_mezzanine_site

Seem familiar?

* This assumes python3 on Ubuntu * python3-dev, libjpeg required for building Pillow * A few other libraries required if you want Pillow to support formats other than PNG + JPEG, see the Pillow docs * Almost identical to getting started with Django

Mezzanine Features

  • Preview unpublished content
  • In-page editing
  • Multi-lingual sites
  • Complete control of markup
  • Disqus, Google Analytics, Akismet integration

Mezzanine Features

Built on top of Django

  • Uses existing Django components where it makes sense
    • Models, forms..
    • Migrations, the ORM...
  • Provides extended version of Django components where it makes things easier
  • Use the existing Django tooling for deploy etc

Mezzanine Features

Structure is seperate from content

  • Structure is your code
  • Edit it with your favorate tools: emacs/Vim/Atom... magnetized needle/butterfly
  • Keep it in your version control system
  • Test it with your CI system
This may not seem paticularly groundbreaking, but if you compare it to Drupal/Wordpress where site structure is in the database with content...

Mezzanine Features

Friendly editing interface

Mezzanine Ecosystem

Haystack

  • Tools for integrating with search backends
  • Mezzanine does have a simple search implementation
  • Haystack is better, but more work to implement
  • haystacksearch.org
  • backends like Solr, Woosh, Elasticsearch
  • Not specifically Mezzanine, doesn't need to be

Mezzanine Ecosystem

django-compressor

  • Static asset concatination + minification
  • Compile your Less/Sass/Coffeescript
  • Mezzanine default template supports compressor
  • You should be using this everywhere...
  • django-compressor.rtfd.org
